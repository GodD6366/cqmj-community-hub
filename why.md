# 为什么做邻里圈

> **一个面向小区真实住户的社区协作平台倡议书 / 白皮书（草案）**
>
> 我们想做的，不是另一个泛论坛，而是一个能真正服务小区生活的公共空间：
> **发需求、发闲置、发帖子、可治理、可协作、可持续演进。**
>
> 技术实现、部署方式和本地运行说明请看 [README.md](./README.md)。

---

## 1. 我们为什么要做这个项目

很多小区里的真实需求，今天仍然依赖微信群、朋友圈、口口相传、公告栏：

- 想找维修、保洁、搬家、家电清洗，信息分散、难检索
- 闲置转让、免费送、以物换物，缺少稳定入口
- 物业通知、邻里反馈、社区讨论，容易被聊天记录淹没
- 住户身份不可信，导致沟通成本高、治理困难大

**邻里圈**希望把这些碎片化信息，变成一个有秩序、有规则、能沉淀价值的社区平台。

它的长期愿景是：

- 让邻里互助更高效
- 让闲置流转更自然
- 让社区公告更清晰
- 让真实住户之间形成更稳定的信任关系
- 让一个小区拥有自己的“社区操作系统”

## 2. 我们和普通论坛的区别

普通论坛通常是“人发帖，人看帖，人处理”。
**邻里圈从一开始就是 AI-native 的。**

这意味着我们不是把 AI 当作一个外挂功能，而是把它当作系统设计的起点：

- 内容发布可以由人完成，也可以由 AI 协助生成和整理
- 帖子分发不只是“展示”，而是会进入自动识别、自动匹配、自动触达流程
- 社区治理不只靠人工审核，也会把规则、标签、分类、处理建议交给 AI 参与分配
- 最终目标是：**人工负责审批和结果确认，日常流转尽量自动化**

### 2.1 先解决最基础的事情

我们优先做的，不是花哨功能，而是最基础、最常见、最有用的能力：

- **发需求**：找服务、找帮助、求组队
- **发闲置**：转让、免费送、以物换物
- **发帖子**：公告、交流、经验分享、意见反馈
- **社区互动**：评论、收藏、举报、治理

### 2.2 AI-native 的工作方式

在这个项目里，AI 不是“可选插件”，而是默认参与者：

- **内容理解**：AI 识别帖子意图、分类、标签、紧急程度
- **需求匹配**：AI 将“需求”与“能力/资源”自动匹配
- **消息触达**：AI 通过微信等渠道自动通知相关用户
- **流程编排**：AI 负责把匹配、通知、审批、结果记录串起来
- **人工只做关键动作**：例如审批、确认结果、处理异常

### 2.3 低门槛参与

这个项目不是只给“会写代码的人”参与的。

我们提倡：

- 没有开发经验的人，也可以参与进来，现在是口喷时代，只要你有想法，代码不是卡点
- 有开发经验的人，可以帮助没有经验的人
- 大家都可以把自己擅长的事情贡献进来，形成一个真正的协作型项目

## 4. 未来希望支持的接入方式

为了让更多人能用、能参与、能扩展，我们希望邻里圈不止有一个网页入口，而是能通过多种方式接入：

### 4.1 Web 访问

这是当前最基础的入口，适合快速浏览、发帖、互动、管理。

### 4.2 MCP / SKILL / CLI 接入

让 AI 助手可以直接在合适的上下文里帮助用户：

- 发现热点
- 了解物业最新动态
- 自动发现“谁能帮忙”“谁需要帮助”

### 4.3 自动匹配与消息触达

这是我们最想做成的能力之一：

- 有人注册一个能力，比如“擅长种花种草”“会修水电”
- 这些能力会被系统结构化登记到平台
- 当其他用户发布的需求和这些能力匹配时，系统自动识别
- 系统自动通过微信触达对应的人或相关用户
- 人工只需要看审批结果，或者在关键节点确认一下

换句话说，**平台不只是一个发帖网站，而是一个自动分配需求的协作网络**。

畅想一下，你的龙虾也可以发布一个技能，当其他龙虾的需求和你的技能匹配后，可以自动完成任务闭环。

### 4.4 WEB / 小程序访问

后续希望支持小程序入口，降低使用门槛，让用户在手机上更自然地参与社区。

### 4.5 微信 iLink 协议 / 微信接入

如果有些人没有环境部署龙虾，我们也可以通过 **微信 iLink 协议** 接入微信，把基础功能带到大家常用的聊天软件中。

这会让更多人“先能用起来”，而不是先被部署门槛挡住。

## 5. 参与方式：不需要你很“专业”

我们希望这个项目成为一个**开放参与**的社区项目，而不是少数人的封闭产品。

### 5.1 非开发者可以做什么

如果你没有开发经验，也完全可以参与：

- 提需求：告诉我们你最需要什么
- 提建议：告诉我们哪里不好用
- 写文案：写社区规则、公告、介绍、活动文案
- 做测试：帮忙试用、提 Bug、提体验问题
- 做运营：组织发布、协助传播、整理用户反馈
- 做产品：帮我们判断什么该先做、什么可以后做

### 5.2 开发者可以做什么

如果你会开发，可以直接参与功能实现：

- 页面与交互
- API 与数据库
- 权限与治理
- 接入 MCP / 小程序 / 微信桥接
- 部署、监控、运维脚本
- 文档和测试

### 5.3 用 AI 一起开发

我们鼓励大家把 AI 当成协作者，而不是门槛：

- 用 AI 写初稿
- 用 AI 生成测试案例
- 用 AI 帮忙理解代码
- 用 AI 把需求变成任务拆解
- 用 AI 降低“我不会写代码”的心理门槛
